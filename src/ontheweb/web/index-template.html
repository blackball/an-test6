Optional Settings

Handedness of your image

Some images have "East" clockwise from "North"; some have the "East" counterclockwise from "North". If you are certain that your image is of one type, you can select it to make the solving process faster.
  • ##parity-both## Try both simultaneously
  • ##parity-right## "Right-handed": East is clockwise from North
  • ##parity-left## "Left-handed": East is counter-clockwise from North

Index to use

We use indexes to solve your fields; an index contains "landmarks" for the sky. The solving process works best when the "landmarks" are a little smaller than the size of the field to be solved. We can automatically choose a suitable index, or you can select one here.
  • ##index##

Star positional error

When we find a matching "landmark", we check to see how many of the stars in your field match up with stars we know about. To do this, we need to know how much a star in your field could have moved from where it should be.
##poserr-err##
  • Positional error, in pixels: ##poserr##

Tweak

After we find a correct match, we usually run a "Tweak" process on it to fine-tune the transformation between field coordinates and sky coordinates.
